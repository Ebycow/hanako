# Discord音声Bot コマンド追加手順書

このドキュメントでは、Discord音声Botプロジェクトに新しいコマンドを追加する完全な手順を説明します。

## アーキテクチャ概要

このプロジェクトはDomain-Driven Design (DDD) パターンを採用し、以下の主要コンポーネントで構成されています：

- **コマンドパターン**: 各コマンドは独立したクラス
- **ファクトリーパターン**: Commandoがコマンドを解決
- **エンティティパターン**: CommandInputでデータ管理
- **サービス層**: MessageServiceが処理を統括

## 新規コマンド追加手順

### 1. コマンドクラスの実装

`src/domain/model/commands/` ディレクトリに新しいコマンドファイルを作成します。

```javascript
// 例: src/domain/model/commands/your_new_command.js
/**
 * 新しいコマンドの実装例
 */
class YourNewCommand {
    constructor(hanako) {
        this.hanako = hanako; // Hanakoモデルへの参照（必要に応じて）
    }

    /**
     * コマンドタイプの定義
     * @returns {string} コマンドタイプ
     */
    get type() {
        return 'your-new-command';
    }

    /**
     * コマンド名の別名リスト
     * @returns {string[]} コマンドの別名配列
     */
    static get names() {
        return ['新コマンド', 'newcmd', 'nc'];
    }

    /**
     * メインの処理ロジック
     * @param {CommandInput} input - コマンド入力情報
     * @returns {Promise<Response>} レスポンス
     */
    async process(input) {
        // 利用可能な情報:
        // input.argc - 引数の数
        // input.argv - 引数の配列
        // input.serverId - DiscordサーバーID
        // input.channelId - DiscordチャンネルID
        // input.userId - DiscordユーザーID
        // input.voiceChannelId - ボイスチャンネルID
        
        // 引数の処理例
        if (input.argc === 0) {
            return input.newChatResponse('引数が必要です');
        }

        const argument = input.argv[0];
        
        // 処理実装
        const result = await this.executeLogic(argument);
        
        // レスポンスの返却
        return input.newChatResponse(`コマンド実行結果: ${result}`);
    }

    /**
     * コマンド固有のロジック
     * @private
     */
    async executeLogic(argument) {
        // 実際の処理を実装
        return `処理完了: ${argument}`;
    }
}

module.exports = YourNewCommand;
```

### 3. スラッシュコマンドとして登録（オプション）

Discordのスラッシュコマンドとしても利用したい場合は、`deploy-commands.js` に定義を追加します：

```javascript
const commands = [
    new SlashCommandBuilder()
        .setName('plz')
        .setDescription(`はなこがボイスチャットを読み上げてくれます`)
        .toJSON(),
    new SlashCommandBuilder()
        .setName('bye')
        .setDescription(`はなこがボイスチャットから退出します．お疲れ！`)
        .toJSON(),
    // 新しいスラッシュコマンド
    new SlashCommandBuilder()
        .setName('yournewcmd')
        .setDescription('新しいコマンドの説明')
        .addStringOption(option =>
            option.setName('parameter')
                .setDescription('パラメータの説明')
                .setRequired(false))
        .toJSON(),
];
```

### 4. アクション実装（状態変更が必要な場合）

コマンドが状態変更を伴う場合は、`src/domain/entity/actions/` にアクションクラスを作成します：

```javascript
// src/domain/entity/actions/your_action.js
/**
 * カスタムアクション
 */
class YourAction {
    constructor(data) {
        this.data = data;
        this.timestamp = new Date();
    }

    get type() {
        return 'your-action';
    }

    // アクション固有のプロパティ
    get actionData() {
        return this.data;
    }
}

module.exports = YourAction;
```

`src/domain/entity/actions/index.js` に登録：

```javascript
module.exports = {
    // 既存のアクション...
    YourAction: require('./your_action'),
};
```

### 7. テストの実装

`test/unit_specs/` にユニットテストを作成します：

```javascript
// test/unit_specs/domain_model_commands_your_new_command_unit_spec.js
const assert = require('assert');
const YourNewCommand = require('../../src/domain/model/commands/your_new_command');
const CommandInput = require('../../src/domain/entity/command_input');

describe('YourNewCommand', () => {
    let command;
    
    beforeEach(() => {
        command = new YourNewCommand();
    });

    describe('type', () => {
        it('should return correct type', () => {
            assert.strictEqual(command.type, 'your-new-command');
        });
    });

    describe('names', () => {
        it('should return command names', () => {
            const expected = ['新コマンド', 'newcmd', 'nc'];
            assert.deepStrictEqual(YourNewCommand.names, expected);
        });
    });

    describe('process', () => {
        it('should process command with arguments', async () => {
            const input = new CommandInput({
                argc: 1,
                argv: ['test'],
                serverId: 'server123',
                channelId: 'channel123',
                userId: 'user123'
            });

            const response = await command.process(input);
            
            assert.strictEqual(response.type, 'chat');
            assert(response.message.includes('test'));
        });

        it('should handle no arguments', async () => {
            const input = new CommandInput({
                argc: 0,
                argv: [],
                serverId: 'server123',
                channelId: 'channel123',
                userId: 'user123'
            });

            const response = await command.process(input);
            
            assert.strictEqual(response.type, 'chat');
            assert(response.message.includes('引数が必要'));
        });
    });
});
```

## レスポンスタイプ

コマンドは以下のレスポンスタイプを返すことができます：

- **ChatResponse**: テキストレスポンス
- **ActionResponse**: 状態変更アクション
- **VoiceResponse**: 音声レスポンス
- **SilentResponse**: 無応答

```javascript
// 使用例
return input.newChatResponse('テキストメッセージ');
return input.newActionResponse(new YourAction(data));
return input.newSilentResponse();
```

## 注意点

1. **命名規則**: ファイル名は `snake_case`、クラス名は `PascalCase`
2. **エラーハンドリング**: 適切な例外処理を実装
3. **テスト**: 新機能には必ずテストを追加
4. **ドキュメント**: JSDocコメントを適切に記述
5. **依存関係**: 必要最小限の依存関係に留める

## トラブルシューティング

### よくある問題

1. **コマンドが認識されない**
   - `index.js` への登録を確認
   - `names` プロパティの設定を確認

2. **スラッシュコマンドが表示されない**
   - `deploy-commands.js` の実行を確認
   - Discord Developer Portalでの設定を確認

3. **テストが失敗する**
   - モックオブジェクトの設定を確認
   - 非同期処理の適切な処理を確認

この手順書に従って実装することで、既存のアーキテクチャに適合した新しいコマンドを確実に追加できます。
